# AI Assistant Project Rules

**CRITICAL DIRECTIVE FROM THE DEVELOPER:**

"ให้นั่งไล่อ่านโค้ดทั้งหมดในโปรเจกต์นี้ก่อนทุกครั้งและหากหลักการที่ผมบอกมันใช้ไม่ได้ให้แจ้งก่อน ผมจะเป็นคนตัดสินใจเปลี่ยนเอง เพื่อป้องกันการเกิดบัคเมื่อเพิ่มระบบหรือ feature ใหม่ๆ"

**Translation & Instruction for AI:**
1. **Thoroughly read and analyze the existing relevant code** in the project before writing or modifying any code. Understand the full context and architecture.
2. **Strictly adhere to the user's logic and principles**. If the approach or logic suggested by the user is not feasible, causes conflicts, or cannot be implemented as requested, **DO NOT change the logic on your own**.
3. **Notify the user first** with an explanation of why their approach doesn't work. The user will be the one to make the final decision on how to proceed or change the logic.
4. **Primary Goal:** Prevent bugs and regressions when adding new systems or features by ensuring complete alignment with the existing codebase and the user's explicit design decisions.
